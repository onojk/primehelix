## Schema

### `semiprime_*_labels.csv`

| Column | Type | Description |
|--------|------|-------------|
| `label` | string | Structure label, e.g. `semiprime \| lopsided \| mod4_1x3` |
| `count` | int | Count of integers with this label in the range |
| `percent` | float | `count / total_semiprimes * 100`, rounded to 4 decimal places |

### `compare_semiprime_*_halves.csv`

| Column | Type | Description |
|--------|------|-------------|
| `structure` | string | Structure label |
| `a_count` | int | Count in the first (lower) half |
| `a_percent` | float | Percent share in the first half |
| `b_count` | int | Count in the second (upper) half |
| `b_percent` | float | Percent share in the second half |
| `delta` | float | `b_percent − a_percent` in percentage points |
| `ratio` | string | `b_percent / a_percent` formatted as `1.11x` |

## Reproduce

Requires `primehelix >= 0.3.0`. Install: `pip install primehelix`

```bash
# 1M label distribution
primehelix structure-scan --start 1 --stop 1000000 \
  --only-classification semiprime \
  --export-csv semiprime_1e6_labels.csv --json

# 1M halves comparison
primehelix compare-ranges \
  --a-start 1 --a-stop 500000 \
  --b-start 500000 --b-stop 1000000 \
  --only-classification semiprime \
  --export-csv compare_semiprime_1e6_halves.csv --json

# 10M label distribution (takes ~10 min)
primehelix structure-scan --start 1 --stop 10000000 \
  --only-classification semiprime \
  --export-csv semiprime_1e7_labels.csv --json

# 10M halves comparison (takes ~10 min)
primehelix compare-ranges \
  --a-start 1 --a-stop 5000000 \
  --b-start 5000000 --b-stop 10000000 \
  --only-classification semiprime \
  --export-csv compare_semiprime_1e7_halves.csv --json
```

Runtime on a modern laptop: ~30 s for 1M, ~10 min for 10M.

### Theory vs measurement

The lopsided fraction is predicted by `primehelix.dickman.theoretical_lopsided_fraction(N)` — a prime-sum model using π(x) ≈ li(x). The measured values match to within 0.2 pp and the gap is shrinking:

| Range | Theory | Measured | Δ (pp) |
|-------|-------:|----------:|-------:|
| 1M    | 72.96% | 73.17%   | +0.21  |
| 10M   | 78.41% | 78.50%   | +0.09  |
| 100M  | 82.16% | 82.22%   | +0.06  |

The delta halves roughly every two orders of magnitude. The bit-gap criterion and the smooth-number model are asymptotically equivalent.
